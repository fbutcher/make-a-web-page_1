NOTES ON IMPORTANT CONCEPTS

Lesson 1: Introduction to the Web and HTML

The World Wide Web

The World Wide Web was created in the 1990's. It is host to approximately 30 billion web pages! The World Wide Web works by users accessing web pages through at browser. The browsers make requests and retrieves page data from web servers via HTTP "Hypertext Transfer Protocol." Servers are computers that store the data on the "backend." What the user sees via the browser is called the "frontend." This video explains in detail how the World Wide Web works.

HTML

HTML - hypertext markup language
HTML was created by Tim Berners-Lee in 1989. This is the language used to write most webpages and is used in conjunction with CSS and JavaScript. HTML is used to display the text of most webpages it is also good at linking images and other webpages. HTML uses tags or elements.

Tags and Elements

HTML Markup uses tags to display hypertext on the web page the following are some examples of tags.

  • "br"--"/br" Allows the user to create a break to move text to the next line.
  • "p"--"/p" A new paragraph is started
  • "em"--"/em" Italics text
  • "b"--"/b" To bold text

It is important to note that each tag is within angle brackets instead of quotes.

A visual example of how tags work can be found here

Inline vs. Block

Block elements create an invisble box within the tags. An example of a block element would be the "p" "/p" tags.
Inline elements such as the "br""/br" tag simply wrap text to the next line.

Computers And Programming

Computers are unable to execute any task on their own. They rely on the user to input instructions before any task can be executed.
For this reason, computers are very unforgiving if the instructions, i.e. the code is not clear and exact. Bad code can result in many hours of problem solving.

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer writes.

Boxes and Tree Structure

Webpages are comprised of boxes. What appears to be a fluid and visual appealing to the user, is nothing more than a page of boxes containing the elements and styles of the visual context. These boxes are formed into a tree structure see this wikipedia page for more context

Lesson 3 Understanding Styles

Styles Upon Styles

HTML structure and syntax is important to build the framework for the project. However, knowing how to style and document with CSS is what allows the coder to create dynamic and eye catching webpages. An analogy would be that HTML is the structure of a house, while CSS is the paint and décor that gives the house a distinct look. This is why coding is sometimes referred to in an artistic sense. Some of the things CSS allows the programmer to are:

  • Add Color
  • Manipulate Various Fonts with Typface, Line Height, and Size
  • Move paragraphs images and text into proper position
  • Build Margins and Borders
What Is CSS?

CSS stands for Cascading Style Sheet. Cascading allows various styles to flow through the document.

  • Author Stylesheets Have Priority
  • Reader Stylesheets Follow
  • Then the Web Browsers Default Style

CSS also allows for separation between the structure and the presentation. This separation is critical because it allows the programmer to avoid repetion in the documents structural content.

Inherited Styles

CSS allows for a parent element/child element relationship. Where the child may inherit some of the style attributes from the parent. However, it is also possible for the child elements own assigned attributes to take precedence over any inherited attributes. These attributes are usually created using the “class” and “id” selectors.

“Class” selectors can be used for assigning elements the same value within a document. “Id” selectors are used for one specific element and that element only.
